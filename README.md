# 个人博客系统

一个简单的个人博客系统，包含前端展示和后台管理功能。

## 主要特点

- 📝 **文章管理**：通过后台管理系统创建、编辑和删除文章
- 📱 **响应式设计**：适配不同屏幕尺寸的设备
- 🎨 **简洁美观**：现代化的UI设计
- 🔄 **自动同步**：后台写完文章后自动同步到博客前端
- 💾 **数据持久化**：文章数据存储在JSON文件中

## 快速开始

### 1. 启动服务器

本博客系统提供了一个自定义的Python服务器，支持自动保存文章功能：

```bash
python server.py
```

服务器启动后，你可以访问：
- 博客首页：http://localhost:8000
- 后台管理：http://localhost:8000/admin.html

### 2. 登录后台

默认管理员密码：`admin123`

你可以在`js/admin.js`文件中修改密码。

### 3. 管理文章

- **创建文章**：点击"新建文章"按钮，填写文章信息后保存
- **编辑文章**：点击文章列表中的文章，修改后保存
- **删除文章**：目前需要通过编辑功能实现

### 4. 自动同步

现在，当你在后台保存文章后，文章会自动同步到`data/articles.json`文件中，博客前端会直接从这个文件加载最新的文章数据。

## 技术栈

- **前端**：HTML, CSS, JavaScript
- **数据存储**：JSON文件
- **服务器**：Python自定义HTTP服务器

## 注意事项

1. **密码安全**：在生产环境中，请务必修改默认密码
2. **数据备份**：定期备份`data/articles.json`文件以防止数据丢失
3. **端口占用**：如果8000端口被占用，可以修改`server.py`文件中的PORT变量

## 自定义配置

### 修改默认作者名

在`js/admin.js`中修改：
```javascript
authorInput.value = '你的名字';
```

### 修改管理员密码

在`js/admin.js`中修改：
```javascript
const ADMIN_PASSWORD = '你的新密码';
```

### 修改服务器端口

在`server.py`中修改：
```python
PORT = 8080  # 新的端口号
```

## 故障排除

### 文章保存失败

如果在后台保存文章时显示同步失败：

1. 确保你使用的是自定义的`server.py`服务器，而不是Python默认的`http.server`
2. 检查控制台是否有错误信息
3. 确认`data`目录是否存在，是否有写入权限

### 前端不显示最新文章

如果前端没有显示最新的文章：

1. 刷新浏览器页面
2. 检查`data/articles.json`文件是否包含最新的文章数据
3. 确保服务器正在运行
